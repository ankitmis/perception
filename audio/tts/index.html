Real TTS never feeds raw text to the model — it first expands currency, dates, ordinals, and abbreviations into spoken words. Highlighted spans changed.

2 Grapheme → phoneme (G2P) eSpeak-NG, in-browser

— IPA phonemes appear here —
Kokoro is driven by phonemes, not letters. These are the actual IPA phonemes (from eSpeak-NG running as WASM in your browser) that the model receives.

82M-param StyleTTS2-style model: an acoustic model predicts the speech representation and a built-in vocoder turns it into 24 kHz audio samples — both stages run on your CPU via ONNX Runtime (WASM).

4 Mel-spectrogram

Computed from the generated audio: STFT → 80-bin mel filterbank → log. x = time, y = mel bin (low→high), color = energy.

5 Waveform & playback

Pipeline: text → normalize → phonemes (eSpeak-NG) → Kokoro acoustic model → vocoder → 24 kHz audio → mel-spectrogram + waveform. Everything bundled in this folder; runs with no network.
