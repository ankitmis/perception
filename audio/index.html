Anatomy of one window: how the FFT & log-mel vector are built

Step 1 · slice × Hann window

Multiply the 400-sample slice by a Hann window (tapers edges to zero so the FFT doesn't leak energy between bins).

Step 2 · zero-pad to 512 & FFT

Pad to 512 (power of 2) so the FFT is fast; keep magnitudes of the first 257 bins.

Step 3 · mel filterbank × power

80 triangular filters span 0–8 kHz on a mel scale; each sums weighted FFT power, then log() compresses it.
Chain for one window: 400 samples → 512 padded → 257 FFT magnitudes → 257 FFT power → 80 mel powers → 80 log-mel values. That 80-vector is one spectrogram column.

Ground truth (reference for WER)

WER = (Substitutions + Insertions + Deletions) / reference-word-count. Case-insensitive, punctuation ignored. Every model runs locally in your browser — no API keys.

Models to compare

All models run fully in your browser via transformers.js — no API key, no API call. Weights are bundled in web-models/ (q8-quantized ONNX) and loaded locally, so it works after cloning even where HuggingFace is blocked. The transformers.js runtime itself loads from a CDN on first use. The first run of each model warms up, then it's fast. Anthropic Claude is listed but can't transcribe audio.
